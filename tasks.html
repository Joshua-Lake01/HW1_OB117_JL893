Homework 1: Tasks for ob117

Your personal task is to develop a system for arcades and their machines.

Task 1: Configuration [4 marks]

Configure the project to support the use of JSPs as view technology. JSPs must be configured to be placed inside folder src/main/webapp/WEB-INF/views/.

Task 2: Domain Classes [13 marks]

Create class co2123.hw1.domain.Machine. With the following private fields:

  1. String manufacturer
  2. String type
  3. String genre
  4. int score
Generate necessary getters and setters. [7 marks]

Create class co2123.hw1.domain.Arcade. With the following private fields:

  1. int id
  2. String name
  3. String email
  4. List<Machine> machines
Generate necessary getters and setters. [6 marks]

Task 3: Controller (simple) [5 marks]

Create controller co2123.hw1.controller.MainController [2 marks].

  1. Requests for URL "/" (POST and GET) should show the view start (the corresponding JSP already exists). [3 marks]

Task 4: Controller, View, and Form [20 marks]

Create field public static List<Arcade> arcades in class co2123.hw1.Hw1Application. Initialize the list inside class co2123.hw1.Hw1Application to contain one arcade with two machines. [4 marks]

Create controller co2123.hw1.controller.ArcadeController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/arcades" should show view arcades/list. [2 marks]
    • Add the list of all arcades from co2123.hw1.Hw1Application to the model as attribute arcades. [2 marks]
    • Create the corresponding JSP to list the name and email of each arcade followed by a link to URL "/machines?arcade={id of arcade}", e.g., "/machines?arcade=3". [4 marks]
  2. Requests for URL "/newArcade" should show view arcades/form. [3 marks] Create the corresponding JSP with a form with modelAttribute="arcade" and action="/addArcade" and inputs for all fields of class Arcade except for machines. [5 marks]
  3. POST requests for URL "/addArcade" should add a new arcade to the list of all arcades in co2123.hw1.Hw1Application and redirect to /arcades. [3 marks]

Task 5: Controller, View, and Form with additional Parameter [33 marks]

Create controller co2123.hw1.controller.MachineController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/machines" should show view machines/list. [2 marks]
    • Based on request parameter arcade determine the corresponding arcade from co2123.hw1.Hw1Application. [2 marks]
    • Add the list of all machines of that arcade to the model as attribute machines. [2 marks]
    • Create the corresponding JSP for the view to list the manufacturer, type, genre, and score of each machine. [4 marks]
    • At the end of the list in the JSP add a link to URL "/newMachine?arcade={id of arcade whose machines are listed}", e.g., "/newMachine?arcade=3". [4 marks]
  2. Requests for URL "/newMachine" should show view machines/form. [3 marks]
    • Create the corresponding JSP with a form with modelAttribute="machine" and action="/addMachine" and inputs for all fields of class Machine. [6 marks]
    • Also add the request parameter arcade with the ID of the current arcade. [4 marks]
  3. POST requests for URL "/addMachine" should add a new machine to the list of machines of the arcade identified by request parameter arcade and then redirect to /arcades. [6 marks]

Task 6: Validation [25 marks]

Set up a validator for class Arcade and make it available in controller ArcadeController. [4 marks]

When adding a new arcade validate that

  1. The value of field id is unique within the list Hw1Application.arcades. [3 marks]
  2. The value of field name is not empty. [2 marks]
  3. The value of field email is not empty. [2 marks]
In case of validation errors go back to the form and show error messages for each field in the form.

Set up a validator for class Machine and make it available in controller MachineController. [4 marks]

When adding a new machine validate that

  1. The value of field manufacturer is not empty. [2 marks]
  2. The value of field type is not empty. [2 marks]
  3. The value of field genre is either "Racing", "Sports", or "Adventure". [3 marks]
  4. The value of field score is between (including) 1 and 5000. [3 marks]
In case of validation errors go back to the form and show error messages for each field in the form.
